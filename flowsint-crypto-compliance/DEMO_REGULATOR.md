# Демо-прототип для госрегулятора (РФ)

Боевой стенд для презентации: **сбор OSINT**, **склейка фиат↔крипто**, **выявление нелегального движения денег и ценностей**.

## Быстрый старт (веб-стенд для презентации)

```bash
cd flowsint
uv sync
uv run flowsint-regulator-stand
```

Откройте **http://localhost:8877** — **боевой прототип ПОД/ФТ**:

| Раздел | Содержание |
|--------|------------|
| **Командный центр** | KPI оперативной сводки · ПОД/ФТ · криптоактивы |
| **Операционный центр** | Очередь проверок с официальными кодами (`STR-SABRRUMM-…`, `MON-PAT-001-…`) |
| **Инструменты ИЦ-01…08** | 8 модулей: хаб 115-ФЗ, монитор паттернов, OSINT fusion, KYT, детектор, отчётность |
| **Реестр отчётов** | Справки по 115-ФЗ с решением и рекомендуемыми действиями |

После расследования автоматически формируется **справка по 115-ФЗ** (ИЦ-08) с правовыми основаниями, признаками ст. 6, решением о направлении в Росфинмониторинг.

Для показа регулятору в той же Wi‑Fi сети используйте ссылку **LAN** на странице (например `http://192.168.x.x:8877`).

## Быстрый старт (CLI)

uv run flowsint-regulator-demo --scenario p2p_rub_offshore

```bash

```

## 4 демо-сценария (реалистичные для РФ)

| ID | Сценарий | Что показывает |
|----|----------|----------------|
| `p2p_rub_offshore` | P2P рубли → OTC hub → Binance (KYT) | STR Сбера + контрольная закупка + мировая база |
| `cis_transit_kz` | РФ → транзит → Казахстан | Коридор СНГ, ВТБ + OTC Telegram |
| `cross_border_do` | Рубли → layering → Доминикана | Трансгран + лицензированный выход |
| `sbp_gray_hub` | СБП → hub fan-out → mixer (KYT) | Прямая склейка банк→адрес + чёрная зона |

## API (полный стенд с PostgreSQL)

```bash
# 1. Поднять Flowsint
docker compose -f docker-compose.dev.yml up -d

# 2. Миграции
cd flowsint-api && alembic upgrade head

# 3. Зарегистрироваться в UI → получить token

# Список сценариев
GET /api/compliance/demo/scenarios

# Мгновенный прогон (без БД)
POST /api/compliance/demo/run/p2p_rub_offshore

# Полный цикл: PostgreSQL + KYT + банки + отчёт
POST /api/compliance/demo/seed/p2p_rub_offshore
```

## Ключевые метрики для презентации

| Метрика | Значение для регулятора |
|---------|-------------------------|
| `illegal_flow_score` | Индекс 0–100 нелегального движения |
| `gray_zone_reduction_pct` | Насколько сузили «серое пятно» |
| `bank_crypto_links` | Склейки банк ↔ крипто |
| `black_zone_wallets` | Чёрная зона (hub/layering) |
| `cross_border_bridges` | Трансграничные мосты |
| `findings[]` | Юридически оформляемые индикаторы |

## Архитектура демо

```
Банки (STR) ──► Хаб регулятора ──► EvidenceGraph
Контрольные закупки ──────────────►     │
KYT bulk (мир) ───────────────────►     ▼
                              OSINTFusionEngine
                                      │
                              IllegalFlowDetector
                                      │
                              RegulatorCaseReport
```

## Суверенность

- **Свой движок** — кластеризация, коридоры СНГ, чёрная зона без Chainalysis
- **Мировые базы** — supplemental через bulk import
- **Банки** — только через хаб регулятора (115-ФЗ), не напрямую

## Следующий шаг после демо

1. Подключить реальный хаб регулятора → `POST /cases/{id}/bank-feeds`
2. Периодический `POST /kyt/import` от интегратора
3. Привязка к расследованиям Flowsint (Neo4j граф)
